# 🆘 Восстановление проекта (новый компьютер / новый чат)

Бэкап бесполезен, если не умеешь восстановить. Вот порядок. Дать новому ИИ эту инструкцию.

## Что где лежит
- **Код + все версии (теги M1, M2, …)** → приватный GitHub репозиторий `labazan78`. Это основная копия.
- **Секреты (`.env`)** → в репозитории их НЕТ. Если появятся — хранить отдельно (на Mac — Google Диск;
  в облаке — создавать заново). Сейчас секретов в проекте нет.
- **Сама система сохранения** → скил/плагин `save-system` (см. §«Система» ниже).

## Восстановление в облачном Claude Code (этот случай)
1. GitHub уже подключён — отдельный `gh auth login` не нужен.
2. Репозиторий клонируется автоматически при старте сессии.
3. Прочитать `НАЧНИ-ОТСЮДА.md`, затем последний снимок в `Сохранение/` → продолжить с §8.
4. Поднять окружение по §5 онбординга (для статического сайта — открыть `index.html`).

## Восстановление на новой локальной машине
1. Поставить инструменты: `git` (+ нужный рантайм). На Mac: `brew install git gh`.
2. Войти в GitHub: `gh auth login` → `gh auth setup-git`.
3. Склонировать: `gh repo clone <владелец>/labazan78` → `cd labazan78`.
4. Секреты: если `.env` появится — взять копию из своего хранилища или пересоздать ключи.
5. Поднять окружение по `НАЧНИ-ОТСЮДА.md` §5.
6. Запустить и проверить в браузере.

## Система сохранения на новой машине
- Скил-папка: скопировать `~/.claude/skills/save-system/` (или распаковать zip из бэкапа).
- Плагин (на Mac): `/plugin` → установить из репо. Встанет скил + команда `/save` одним действием.

> Раз в пару месяцев полезно сделать пробное восстановление в отдельную папку —
> убедиться, что бэкапы реально разворачиваются.
