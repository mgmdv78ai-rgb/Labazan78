# КАК СОХРАНЯТЬ ПРОЕКТ

> Это **универсальная инструкция «как сохранять»** — одна на все проекты.
> Она объясняет ИИ-ассистенту (Claude), как сохранять прогресс, переносить
> проект в новый чат и откатываться назад через git.
>
> Для **продолжения** конкретного проекта используется другой файл —
> `НАЧНИ-ОТСЮДА.md` (создаётся автоматически, своя копия в каждом проекте).

---

## 🎯 Зачем это нужно

Чат с ИИ не вечный: окно закрывается, контекст теряется. Чтобы проект
не пропал и его можно было продолжить в любом новом чате (и даже на другом
компьютере), мы храним его в **приватном репозитории GitHub** и поддерживаем
файл `НАЧНИ-ОТСЮДА.md`, в котором всегда записано, на каком мы шаге.

Правило одной строкой:
- `КАК-СОХРАНЯТЬ-ПРОЕКТ.md` = «инструкция, как сохранять» (этот файл).
- `НАЧНИ-ОТСЮДА.md` = «на чём мы остановились» (точка продолжения).

---

## 💾 Как сохранить прогресс (делает ИИ)

Когда пользователь говорит **«Сохрани проект»**, ИИ должен:

1. **Обновить `НАЧНИ-ОТСЮДА.md`** — записать текущий шаг, что сделано,
   что дальше, важные решения и пути к ключевым файлам.
2. **Закоммитить** все изменения с понятным сообщением:
   ```bash
   git add -A
   git commit -m "Сохранение: <короткое описание шага>"
   ```
3. **Запушить** в приватный репозиторий:
   ```bash
   git push -u origin <имя-ветки>
   ```
   При сетевых сбоях повторить до 4 раз с задержкой 2с, 4с, 8с, 16с.
4. **Сообщить пользователю**: на каком мы шаге и что сохранено.

> ⚠️ Не создавать Pull Request без явной просьбы пользователя.

---

## 🔄 Как продолжить проект в новом чате

### На этом же компьютере
Пользователь пишет:
```
Прочитай файл <путь к проекту>/НАЧНИ-ОТСЮДА.md и следуй ему. Скажи, на каком мы шаге.
```

### На другом компьютере
Пользователь пишет:
```
Склонируй мой приватный репозиторий <ссылка на GitHub>, потом прочитай в нём НАЧНИ-ОТСЮДА.md и следуй ему.
```

ИИ читает `НАЧНИ-ОТСЮДА.md`, восстанавливает контекст и продолжает с того
же места.

## ⏪ Как откатиться назад (git rollback)

Каждое сохранение — это коммит. Чтобы вернуться к прошлому состоянию:

1. Посмотреть историю:
   ```bash
   git log --oneline
   ```
2. Вернуться к нужному коммиту **без потери истории** (безопасно):
   ```bash
   git revert <хеш-коммита>
   ```
3. Либо жёстко откатить рабочую копию к коммиту (осторожно, теряются
   незакоммиченные изменения):
   ```bash
   git checkout <хеш-коммита> -- .
   ```
4. Полный жёсткий откат ветки (только по явной просьбе пользователя):
   ```bash
   git reset --hard <хеш-коммита>
   ```

> ⚠️ `reset --hard` и `push --force` ИИ применяет только с явного
> разрешения пользователя — они переписывают историю.

## ✅ Чек-лист для ИИ при команде «Сохрани проект»

- [ ] Обновил `НАЧНИ-ОТСЮДА.md` (шаг, сделано, дальше).
- [ ] `git add -A && git commit` с понятным сообщением.
- [ ] `git push -u origin <ветка>` (с ретраями при сбоях).
- [ ] Сообщил пользователю текущий шаг и факт сохранения.
- [ ] Pull Request НЕ создавал (если не просили).
